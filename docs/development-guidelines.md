# development-guidelines.md

코드 수정 시 이슈/브랜치/커밋 관련 기본 규칙을 정의합니다.

## 지침

- 코드에 변경이 필요하면, 먼저 GitHub CLI를 이용하여 관련 이슈가 있는지 확인한다.
- 관련 이슈가 없다면, 새 이슈를 생성한 뒤 해당 이슈를 기준으로 작업을 진행한다.
- 관련 이슈가 있다면(또는 새로 만들었다면), `develop` 브랜치를 base로 새 브랜치를 생성한다.
- 브랜치명은 `feat/#이슈번호-짧은영어요약` 형식을 따른다. (예: `feat/#123-fix-login-bug`)
- 모든 작업은 해당 feature 브랜치에서 진행한다. (직접 `develop`에 작업 금지)
- 하나의 feature(의미 있는 단위 작업)가 끝날 때마다 커밋한다.

추가 표준(명명/구조/테스트/환경변수 등)은 `docs/development-standards.md`를 따른다.
